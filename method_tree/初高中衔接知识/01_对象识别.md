# 对象识别

对象识别是方法树的入口。很多题目看起来都是一串符号，但同一个式子在不同位置承担的任务不同。

例如：

```text
x^2 - 4x + 3 = 0
```

这是方程，目标是找哪些 `x` 让等式成立。

```text
y = x^2 - 4x + 3
```

这是函数对象，目标可能是研究零点、正负、顶点、变化趋势或定义域。

```text
x^2 - 4x + 3
```

这是表达式，目标可能是展开、分解、配方、化简或比较。

## 第一步：判断对象身份

| 看到的形式 | 先判断为 | 继续追问 |
| --- | --- | --- |
| 只有一个式子 | 表达式 | 要展开、分解、化简，还是看结构？ |
| 左右有等号 | 方程 | 哪些值让左右相等？ |
| 有大于、小于、不等号 | 不等式 | 哪些范围让关系成立？ |
| 写成 `y=f(x)` | 函数对象 | 零点、正负、图像、定义域和值域是什么？ |
| 含分母、根号 | 带条件表达式 | 原式允许哪些值？ |
| 含参数 | 对象族 | 参数改变时结构怎样变化？ |

## 第二步：判断当前目标

对象身份确定后，再看题目目标。

| 目标 | 常见方法入口 |
| --- | --- |
| 看等于 `0` 的位置 | 因式分解、零点判断 |
| 看正负 | 分解后分区间、符号表 |
| 看平方结构 | 完全平方、配方 |
| 看乘法来源 | 因式分解 |
| 化简表达式 | 原式条件、分解、约分、根式非负 |
| 解方程 | 变形、候选解、验回原式 |
| 解不等式 | 变成与 `0` 比较、零点禁点分区间 |

## 第三步：不要混淆对象

常见混淆包括：

| 混淆 | 正确判断 |
| --- | --- |
| 把 `(a-b)^2` 当成 `a^2-b^2` | 前者是差整体平方，后者是两个平方相减 |
| 把 `(a+b)^3` 当成 `a^3+b^3` | 前者要展开，后者要分解 |
| 把化简后的式子当成原式 | 化简后还要保留原式条件 |
| 把候选解当成最终答案 | 候选解必须经过条件筛选和代回检查 |

## 判断句式

做题前可以先写一句判断：

```text
这个式子现在是 ______。
题目要我看 ______。
所以我先检查 ______。
```

例如：

```text
这是一个含分母的表达式。
题目要化简。
所以我先写原分母不为 0 的条件，再分解约分。
```

## 检查点

对象识别完成后，至少要明确三件事：

1. 当前对象是什么。
2. 当前目标是什么。
3. 有没有条件或解集风险。
