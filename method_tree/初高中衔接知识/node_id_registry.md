# 初高中衔接方法树与练习树 ID 注册表

这个文件固定方法节点、练习节点和核心错因的稳定 ID。后续题库、反馈、页面跳转和学习路径推荐都应优先引用这些 ID。

规则：

```text
已经使用的 ID 不重命名、不复用。
如果需要新增节点，只能追加新的 ID。
中文标题可以调整，ID 保持稳定。
```

## 方法节点

| ID | 节点 | 对应文件 | 解决的问题 |
| --- | --- | --- | --- |
| `M_BRIDGE_001` | 对象识别 | `01_对象识别.md` | 先判断当前对象是表达式、方程、不等式、函数还是带条件表达式 |
| `M_BRIDGE_002` | 公式与结构判断 | `02_公式与结构判断.md` | 判断公式对象是否匹配，以及当前要展开还是分解 |
| `M_BRIDGE_003` | 多项式变形与因式分解 | `03_多项式变形与因式分解.md` | 根据结构信号选择提公因式、公式法、十字相乘、分组或配方 |

## 核心错因 ID

核心错因 ID 用于题库诊断和学习路径回退。更细的 `error_tags` 仍保留在 `common_error_tags.md` 中，用于具体反馈。

| ID | 核心错因 | 常见细标签 | 典型反馈方向 |
| --- | --- | --- | --- |
| `E_BRIDGE_001` | 整体平方误当分别平方 | `missing-cross-term` | 先写成两个相同括号相乘，检查交叉项 |
| `E_BRIDGE_002` | 平方差与差的平方混淆 | `object-mix`, `difference-required` | 判断是整体平方，还是两个平方相减 |
| `E_BRIDGE_003` | 因式分解方法选择错误 | `common-factor-missed`, `constant-only-factor`, `cross-product-error` | 先看结构信号，再选方法，最后乘回检查 |
| `E_BRIDGE_004` | 约分后丢失禁点 | `domain-lost` | 被约掉的分母因子仍然留下原式禁点 |
| `E_BRIDGE_005` | 根式非负性错误 | `sqrt-absolute-missed` | `sqrt(A^2)` 先写成 `|A|`，不能机械写成 `A` |
| `E_BRIDGE_006` | 方程增根未验 | `candidate-as-answer`, `extraneous-root` | 候选解必须满足原式约束并代回检查 |
| `E_BRIDGE_007` | 不等式端点或禁点错误 | `endpoint-error`, `forbidden-point-included` | 分子零点和分母禁点的取舍规则不同 |
| `E_BRIDGE_008` | 复合条件遗漏 | `compound-condition-missed` | 根号、分母、约分条件要同时保留并取交集 |
| `E_BRIDGE_009` | 因子与项混淆 | `factor-term-cancel` | 只能约共同因子，不能约加减式中的某一项 |
| `E_BRIDGE_010` | 化简后扩大原式范围 | `range-expanded` | 化简式有值不代表原式也有值，必须比较原式条件 |

## Chapter 04 样板闭环

第一批样板闭环固定为：

```text
Chapter 04 分式根式与条件控制
→ M_BRIDGE_004 条件控制与等价化简
→ P_BRIDGE_004 条件检查练习
→ E_BRIDGE_004 / E_BRIDGE_005 / E_BRIDGE_008 / E_BRIDGE_009 / E_BRIDGE_010
```

这个闭环先验证“表达式形式 + 成立条件”是否能从知识点进入方法、练习、错因和反馈。
