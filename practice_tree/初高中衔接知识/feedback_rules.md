# 反馈规则

反馈的目标不是只告诉学习者答案，而是指出他在哪一层判断出了问题。

## 反馈基本格式

```text
你现在的问题属于：______
错在：______
应该先检查：______
正确思路是：______
```

## 分层反馈

| 错误层级 | 反馈重点 |
| --- | --- |
| 对象识别 | 说明当前对象身份和任务目标 |
| 结构判断 | 指出结构信号是否匹配方法 |
| 方法执行 | 指出计算步骤中哪个关系没有对上 |
| 条件检查 | 指出原式限制、禁点、非负性或端点遗漏 |
| 综合迁移 | 指出是哪一层导致最终答案错误 |

## 公式类反馈

不要只说“公式错了”。要说明：

```text
这个公式处理的对象是什么。
你的式子是否匹配这个对象。
如果是展开，漏了哪些项。
如果是分解，乘回去哪里不能抵消。
```

示例：

```text
你把 (a+b)^2 当成 a^2+b^2 了。
括号平方表示两个相同括号相乘，中间会出现 a·b 和 b·a 两个交叉项。
```

## 因式分解反馈

要优先检查：

```text
有没有先看公因式。
方法是否匹配结构。
十字相乘的一次项是否对上。
分组后有没有共同括号。
能不能乘回原式。
```

示例：

```text
常数项对了，但一次项没有对上。
因式分解二次三项式时，不只要两个数相乘等于常数项，还要交叉项合成原来的一次项。
```

## 条件控制反馈

要强调“原式条件”：

```text
这个值不是化简后才出现的问题，而是原式一开始就不允许。
约分只改变表达式外形，不能把原分母禁点带回来。
```

示例：

```text
你忘记了被约掉的 x-2。
虽然化简式在 x=2 时有值，但原分母在 x=2 时为 0，所以 x=2 不能进入答案。
```

## 方程不等式反馈

要区分候选结果和最终解集：

```text
你算出的只是候选解。
它必须满足原方程条件。
如果变形中平方、去分母或约掉含未知数因子，就要代回原式检查。
```

不等式反馈要说明：

```text
乘除负数为什么翻号。
为什么不能直接除以符号不确定的式子。
零点和禁点在数轴上的作用不同。
```

## 反馈避免事项

不要只写：

```text
答案错误。
重新计算。
注意符号。
```

应该写：

```text
哪个结构判断错了。
哪个条件漏了。
哪个步骤改变了解集。
应该用什么检查方法发现错误。
```
